# Event Request Sentiment Analysis

This project analyzes an event management dataset and assigns a sentiment result to each event request. It includes two complementary approaches: a transparent rule-based scorer and an NLP-based VADER sentiment analyzer. Both workflows load the source CSV, calculate a sentiment score from `-100` to `100`, explain the reason behind the score, export enriched results, and provide Plotly visualizations.

## Dataset

The source file is `Data/EventManagmentDataSet.csv`. It contains event booking records with fields such as:

- `Event`
- `Registers`
- `Type`
- `Request for Booking`
- `Request for Cancelling`
- `Room Available/Not Available`
- `Accept/Reject`

Generated output files:

- `Data/event_sentiment_analysis.csv`: rule-based explainable sentiment output
- `Data/event_nlp_sentiment_analysis.csv`: NLP sentiment output using VADER `SentimentIntensityAnalyzer`

## What The Analysis Does

The rule-based sentiment analysis adds three columns to the original dataset:

- `Sentiment Score`: numeric score between `-100` and `100`
- `Sentiment Label`: `Positive`, `Neutral`, or `Negative`
- `Sentiment Reason`: plain-language explanation for why the row received that score

The NLP sentiment analysis adds additional VADER-based columns:

- `NLP Text`: natural-language text generated from each event row
- `NLP Negative`: VADER negative probability-style score
- `NLP Neutral`: VADER neutral probability-style score
- `NLP Positive`: VADER positive probability-style score
- `NLP Compound`: VADER compound score from `-1` to `1`
- `NLP Sentiment Score`: compound score scaled to `-100` to `100`
- `NLP Sentiment Label`: `Positive`, `Neutral`, or `Negative`
- `NLP Sentiment Reason`: explanation based on the strongest sentiment-bearing text clauses

Example interpretation:

- A rejected event with an unavailable room receives a negative score because the request failed and the required resource was not available.
- An accepted event with an available room and high importance receives a positive score because the request outcome is successful and the event has strong priority signals.

## Scoring Logic

### Rule-Based Scoring

The sentiment score is rule-based and transparent. Each event starts at `0`, then receives positive or negative adjustments based on the event request details.

Positive signals include:

- Request was accepted
- Room was available
- Event was marked `Important` or `Very important`
- Booking was requested
- Registration count was strong or very high
- Event title contained positive activity keywords such as `creative`, `webinar`, `quiz`, `debate`, `review`, or `contest`

Negative signals include:

- Request was rejected
- Room was not available
- Cancellation was requested
- Registration count was low

The final score is capped between `-100` and `100`.

### NLP Scoring With VADER

The NLP notebook uses `SentimentIntensityAnalyzer()` from the `vaderSentiment` package. This is NLP because it analyzes natural-language text and produces sentiment scores from language patterns.

VADER is a lexicon and rule-based NLP sentiment analyzer. It is not a deep-learning model like BERT or RoBERTa, and it does not train on this dataset. Instead, it uses a sentiment dictionary plus linguistic rules for things like negation, intensity, punctuation, and capitalization.

Since the original dataset is structured, each row is first converted into natural-language text such as:

```text
The event request is for Webinar on Web-Technologies.
The event priority is Normal.
The event has 77 registrations.
The room is Not Available.
The request decision is Reject.
```

VADER then scores this generated text and returns `negative`, `neutral`, `positive`, and `compound` values. The `compound` value ranges from `-1` to `1`; the notebook scales it to `-100` to `100` so it can be compared with the rule-based score.

The NLP reason is generated by scoring the individual text clauses and selecting the strongest positive, negative, or neutral clauses behind the final label.

## Notebooks

### `Script/sentiment_analysis.ipynb`

Main rule-based analysis notebook. It loads the event dataset, applies the deterministic scoring logic, displays the enriched table, and writes `Data/event_sentiment_analysis.csv`.

### `Script/nlp_sentiment_analysis.ipynb`

NLP analysis notebook. It converts every event row into natural-language text, applies VADER `SentimentIntensityAnalyzer()`, explains the NLP result, creates dark-themed Plotly charts, and writes `Data/event_nlp_sentiment_analysis.csv`.

### `Script/sentiment_analysis_plots.ipynb`

Visualization notebook for the rule-based results. It reads `Data/event_sentiment_analysis.csv` and shows interactive Plotly charts using a dark theme.

Charts included:

- Sentiment label distribution
- Sentiment score by event
- Distribution of sentiment scores
- Sentiment score vs. registration count

## Streamlit App

`streamlit_app.py` provides an interactive user interface for comparing the two sentiment methods without editing notebooks.

The app supports:

- Reviewing the existing dataset with both rule-based and VADER NLP scores
- Entering a new event request manually
- Running both sentiment methods on the new entry
- Comparing scores, labels, and reasons side by side
- Viewing Plotly charts for method-level comparison

The app uses shared scoring functions from `Script/sentiment_methods.py`, so the logic is reusable outside the notebooks.

### App Tabs

The Streamlit app has three main tabs:

- `New Entry`: enter a new event request and compare rule-based vs. VADER NLP sentiment results.
- `Dataset Comparison`: inspect existing dataset rows with both method outputs side by side.
- `Charts`: view interactive Plotly charts comparing scores, labels, and agreement patterns.

### Run The App

Install dependencies:

```bash
python3 -m pip install -r requirements.txt
```

Run the app from the project root:

```bash
streamlit run streamlit_app.py
```

Then open the local Streamlit URL shown in the terminal, usually:

```text
http://localhost:8501
```

## Notes

This project now contains two complementary approaches:

- A deterministic rule-based method that is easy to audit and directly tied to the structured dataset columns.
- An NLP-based VADER method that analyzes generated natural-language text using lexicon and linguistic rules.

For a larger real-world project, the next improvement would be to validate both methods against human-labeled examples and compare them with a transformer-based sentiment model or an LLM-based evaluator.
